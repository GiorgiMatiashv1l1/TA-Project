Final Group Project: End-to-End Test Automation
Framework
Target Application
Frontend (UI): https://automationexercise.com
UI Test Case Reference: https://automationexercise.com/test_cases
Backend (API): https://automationexercise.com/api_list

Project Scope and Requirements
Students are required to develop a comprehensive End-to-End Test Automation Framework
covering both UI and API testing for the Automation Exercise platform.
The project must meet the following requirements:
1. Test Case Selection
   ○ Select 10 test cases from the official UI test cases page.
   ○ Select 10 test cases from the official API list.
   ○ The total number of automated test cases must be 20.
   ○ Only existing and valid test cases from the provided links may be used.
2. Automation Stack (Mandatory)
   ○ Java
   ○ Selenium WebDriver
   ○ TestNG
   ○ RestAssured
   ○ Maven
   ○ Allure Reporting
   ○ Page Object Model (POM)
3. Framework Design
   ○ The project must follow the Page Object Model (POM) design pattern.
   ○ UI and API tests must be clearly separated.
   ○ The framework must support parallel test execution using TestNG.
   ○ Explicit waits must be used; hard-coded waits are not permitted.
4. Reporting
   ○ All tests must generate Allure reports.
   ○ Reports must include:
   ■ Test steps
   ■ Screenshots for failed UI tests
   ■ API request and response attachments

5. Group Work Requirements
   ○ Maximum group size: 5 students
   ○ Each group member must be responsible for specific test cases.
   ○ Work must be evenly distributed across the team.
6. Repository Requirements
   ○ The project must be hosted in a public GitHub repository.
   ○ The repository must include a README.md file containing:
   ■ Names of all group members
   ■ A clear mapping of which test cases were automated by each member
   ■ Instructions for running the tests and generating reports

7. Submission
   ○ The GitHub repository URL must be shared via email.
   ○ No ZIP files, screenshots, or private repositories are allowed.
